# 《僵尸狂潮：超载重启》大作业报告

## 1. 项目概述

本项目是一款 2D 俯视角 Roguelite 生存射击游戏。玩家在僵尸潮中移动、自动射击、收集经验水晶升级，并通过“主动武器 + 被动技能”的组合触发超进化武器。单局结束后，玩家获得局外金币，用于提升下一局开局属性，形成“局内成长 + 局外养成”的循环。

## 2. 核心玩法

游戏核心循环为：

```text
移动走位 -> 自动攻击 -> 击杀敌人 -> 收集经验/金币 -> 升级选择 -> 武器成型 -> 挑战 Boss -> 结算强化
```

局内限制为 3 个主动武器槽和 3 个被动技能槽。槽位满后，升级池会优先给已有武器/被动的升级项，避免玩家继续选择超出槽位的新技能。武器达到 5 级且拥有对应被动后，可以触发超进化；超进化后会释放对应被动槽位，但被动属性仍然保留。

## 3. 已实现系统

- 玩家移动、鼠标瞄准、相机跟随、准心和虚线瞄准线。
- 六种主动武器及升级：手枪、霰弹枪、电磁手套、黑洞、光刃、激光。
- 六种超进化武器，每种都有强化后的攻击效果和视觉表现。
- 十二种被动技能，包含输出、生存、拾取、经济和复活方向。
- 敌人波次系统，支持普通敌、远程敌、坦克敌、中期 Boss 和最终 Boss。
- 经验水晶、金币、补给箱、回血、磁铁吸取、炸弹清屏。
- 主菜单、暂停菜单、升级选择、结算面板和局外永久强化。
- 程序化像素风角色、敌人、道具、技能图标和音效。
- Unity 对象池，降低大量敌人、子弹和掉落物带来的运行开销。

## 4. 技术实现要点

### 4.1 场景自动生成

项目使用编辑器脚本生成主场景、预制体和程序化贴图，减少手动搭场景造成的遗漏。核心入口位于：

```text
Game/Assets/Editor/ZombieOverdriveSceneBuilder.cs
```

编辑器菜单中可运行：

```text
Zombie Overdrive/Build Prototype Scene
Zombie Overdrive/Smoke Test Prototype
Zombie Overdrive/Validate Prototype Scene
```

### 4.2 武器与升级

每个武器都是独立组件，继承统一的武器基类。升级系统负责维护当前武器、被动、槽位、等级和超进化条件。超进化后不仅修改数值，也会改变攻击形态，例如：

- 手枪变为多发扇形高速穿甲弹。
- 霰弹枪变为大范围爆裂弹幕。
- 电磁手套变为双主链雷暴。
- 黑洞变为多重暗物质涡旋。
- 光刃变为 360 度剑阵。
- 激光变为三线分束切割。

### 4.3 UI 与菜单

UI 全部使用中文。主菜单是独立全屏页面；局内 HUD、暂停菜单、升级选择和结算面板使用统一暗色面板风格。暂停菜单可以查看当前主动武器、被动技能、等级、击杀数和超进化搭配。

### 4.4 局外养成

局外金币通过击杀、补给和结算获得。主菜单中提供永久天赋升级，包括生命、移速、武器伤害、拾取范围、金币倍率和复活能力。局外养成让开局更稳，但中后期敌人会通过血量、伤害、数量和 Boss 压力追上玩家。

## 5. 平衡设计

当前平衡方向是：

- 无局外强化时可以完成前期体验，但不会过快成型。
- 有局外强化后，开局更轻松，适合反复挑战。
- 中后期敌人强度明显提升，要求玩家利用超进化和补给资源。
- 电磁手套已加强链式伤害和超进化雷暴表现，避免弱于激光太多。
- 霰弹枪降低单颗弹丸收益，保留近距离爆发特色。

## 6. 美术与音效

项目未依赖外部素材，当前素材由工程内程序化生成。角色采用简化像素模型，敌人根据血量切换受伤/濒死贴图。主动武器和被动技能都有独立图标。音效同样由程序生成，覆盖射击、拾取、升级、受伤、超进化、胜利和失败。

## 7. 测试与构建

已准备以下自动验证方式：

```powershell
.\scripts\build-windows.ps1
.\scripts\smoke-build.ps1
```

编辑器内验证包括：

```text
ZombieOverdriveSceneBuilder.SmokeTestPrototype
ZombieOverdriveSceneBuilder.ValidatePrototypeScene
```

Windows 构建输出默认为：

```text
Builds/Windows/ZombieOverdrive.exe
```
