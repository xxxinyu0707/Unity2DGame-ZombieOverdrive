# 《僵尸狂潮：超载重启》设计说明

## 1. 项目基本信息

**项目名称**：僵尸狂潮：超载重启  
**开发工具**：Tuanjie / Unity  
**游戏类型**：2D 俯视角 Roguelite 生存射击  
**运行平台**：Windows 单机  
**提交内容**：Unity 游戏工程文件夹、编译后的 Windows 可执行文件、设计说明 PDF 文档

本项目是一款以“僵尸潮生存、局内成长、局外强化”为核心的单机游戏。玩家在无限循环的战场中移动、瞄准、自动攻击，击杀敌人后收集经验和金币，通过升级选择构筑武器组合，并在 Boss 压力下尽可能完成一局挑战。单局结算获得的金币会保留到局外，用于提升下一局的初始属性，形成反复挑战和逐步变强的循环。

## 2. 游戏使用说明

### 2.1 启动方式

解压提交的 Windows 游戏压缩包后，双击：

```text
ZombieOverdrive.exe
```

即可运行游戏。运行时需要保持以下文件和文件夹与 exe 位于同一目录：

```text
ZombieOverdrive_Data/
MonoBleedingEdge/
UnityPlayer.dll
UnityCrashHandler64.exe
```

### 2.2 基本操作

| 操作 | 功能 |
| --- | --- |
| `W / A / S / D` | 控制角色移动 |
| 鼠标移动 | 控制瞄准方向 |
| `ESC` | 暂停游戏，查看当前武器、被动、超进化搭配和音量设置 |
| 鼠标左键 | 点击 UI 按钮、选择升级词条 |
| 升级界面“刷新词条” | 一局内最多刷新 3 次升级选项 |

游戏采用自动攻击机制。玩家主要负责走位、拾取经验、规划升级路线和躲避敌人。

### 2.3 游戏目标

玩家需要在持续增强的僵尸潮中存活，并通过升级武器和被动技能逐步形成战斗力。游戏中会出现普通敌人、特殊敌人、中期 Boss 和最终 Boss。击败敌人可以获得经验、金币和补给资源。单局结束后，根据击杀数、局内金币和胜利状态获得局外金币。

### 2.4 菜单说明

主菜单包含“永久强化”系统。玩家可以消耗金币提升生命、移动速度、武器伤害、拾取范围、金币收益和复活能力。暂停菜单可以查看当前主动武器、被动技能、作战信息和超进化搭配，同时可以调节音乐音量、继续游戏、重开或退出到主菜单。

## 3. 游戏设计思路

### 3.1 核心体验目标

本游戏希望实现一种简单但完整的 Roguelite 生存体验：玩家初期较弱，需要通过走位和拾取经验建立第一套武器组合；中期开始出现特殊敌人和 Boss，要求玩家进行升级路线规划；后期敌人数量、血量和攻击压力提升，玩家需要依靠超进化武器、补给箱和局外养成完成挑战。

设计重点包括：

- **低操作门槛**：攻击自动进行，玩家只需移动和瞄准。
- **高成长反馈**：武器每一级升级都能带来实际效果变化。
- **组合构筑感**：主动武器与对应被动满级后可以触发超进化。
- **局内外循环**：失败也能获得金币，用于下一局永久强化。
- **清晰可展示**：所有核心系统都有 UI、特效、音效或菜单反馈。

### 3.2 核心循环

游戏的主要循环如下：

```text
进入主菜单
  -> 使用局外金币强化角色
  -> 开始战斗
  -> 移动、瞄准、自动攻击
  -> 击杀敌人并收集经验/金币
  -> 升级三选一，规划武器和被动
  -> 主动武器 + 对应被动满级后触发超进化
  -> 挑战更高强度敌人和 Boss
  -> 死亡或胜利结算
  -> 获得局外金币，回到主菜单继续强化
```

## 4. 主要系统设计

### 4.1 玩家与战斗系统

玩家角色采用 2D 俯视角移动方式，通过 `WASD` 控制移动，鼠标控制瞄准方向。游戏中会显示准心和虚线瞄准线，帮助玩家判断攻击方向。武器会自动按照冷却时间攻击，避免玩家在大量敌人围攻时需要频繁点击。

为了增强可读性，项目对不同攻击类型做了区分：

- 手枪子弹、霰弹弹丸和分裂弹使用不同形状和视觉表现。
- 电磁手套使用更粗的电弧表现链式攻击。
- 光刃拥有明显的挥剑弧线和剑阵特效。
- 黑洞使用旋转涡旋、吸积盘和吸入粒子表现范围吸附。
- 激光使用持续光束和超进化后的多线切割表现。

### 4.2 武器升级系统

游戏中共有六种主动武器：

| 主动武器 | 基础定位 | 升级特点 | 超进化方向 |
| --- | --- | --- | --- |
| 哨兵手枪 | 稳定单体输出 | 射速、伤害、分裂弹、双枪 | 多发高速穿甲弹 |
| 爆裂霰弹枪 | 近距离爆发 | 弹丸数量、集中度、燃烧区、击退 | 大范围爆裂弹幕 |
| 电磁手套 | 链式清怪 | 连锁数量、减速、眩晕、分叉 | 双主链雷暴 |
| 重力黑洞 | 范围控制 | 吸附范围、持续伤害、吞噬弹体 | 多重暗物质涡旋 |
| 光刃影切 | 近战防线 | 斩击角度、剑气、吸血和斩杀 | 360 度剑阵 |
| 裂变激光枪 | 持续穿透 | 加粗、折射、热能过载 | 三线分束切割 |

每种武器都有 1 到 5 级的实际机制成长，而不是只提升数值。例如手枪会获得分裂弹，霰弹枪命中后会留下燃烧区域，黑洞可以出现双黑洞，激光可以折射，光刃会扩大斩击范围。

### 4.3 被动技能与超进化

游戏限制为 3 个主动武器槽和 3 个被动技能槽。槽位满后，升级池会优先提供已有武器和已有被动的升级项，避免继续选择超出槽位的新技能。

超进化条件为：

```text
主动武器达到 5 级 + 对应被动技能达到 5 级
```

触发超进化后，武器会获得明显更强的攻击形态和特效变化。对应被动槽会被释放，但已获得的被动属性仍然保留，这样既体现了构筑奖励，也给玩家后续继续选择其他被动留下空间。

### 4.4 敌人与 Boss 系统

敌人系统包含多种类型：

| 敌人类型 | 特点 |
| --- | --- |
| 普通僵尸 | 基础近战追击敌人，用于形成主要尸潮 |
| 快速僵尸 | 移动速度更快，压迫玩家走位 |
| 喷吐僵尸 | 远程发射酸液弹，迫使玩家改变移动路线 |
| 坦克僵尸 | 血量更高，承担前排压力 |
| 中期 Boss | 具有冲锋、地裂等技能 |
| 最终 Boss | 具有血弹、酸雨、狂暴等更高压力技能 |

敌人会根据血量切换正常、受伤和濒死贴图，使玩家可以通过外观判断敌人状态。Boss 技能带有预警表现，降低“突然受伤”的挫败感。

### 4.5 经验、补给与可破坏物

敌人死亡后会掉落经验水晶和少量金币。玩家靠近经验水晶后会自动吸取，拾取范围可以通过局内被动和局外天赋提升。

地图中会生成可破坏补给箱。补给箱被武器破坏后，有概率掉落以下资源：

- **金币**：增加本局收益。
- **烤鸡**：回复生命值。
- **磁铁**：吸取全屏范围内的经验和资源。
- **炸弹**：对大范围敌人造成伤害或清场。

补给箱、炸弹和燃烧区域都使用内部填充型特效，避免只显示单一圆圈，让视觉表现更贴近像素动作游戏。

### 4.6 局外养成系统

局外养成位于主菜单“永久强化”区域。玩家可以花费金币升级六项能力：

| 永久强化 | 效果 |
| --- | --- |
| 强化体质 | 提升生命上限 |
| 机动训练 | 提升移动速度 |
| 武器校准 | 提升全武器伤害 |
| 磁场背包 | 提升初始拾取范围 |
| 收益芯片 | 提升金币收益 |
| 备用生命 | 解锁复活并提升复活后的生命 |

局外养成的设计目标是让玩家下一局开局更轻松，但不会直接破坏中后期难度。中后期通过固定时间推进的怪物血量、刷怪速度和敌人组合来维持压力。

## 5. 业务流程说明

从软件流程角度看，本游戏可以分为“主菜单流程”“单局战斗流程”“升级流程”“结算流程”四部分。

### 5.1 主菜单流程

```text
启动游戏
  -> 加载主菜单
  -> 读取局外金币和天赋等级
  -> 玩家选择永久强化或直接开始
  -> 点击开始战斗
  -> 进入局内状态
```

主菜单不仅是入口页面，也承担局外养成、音量调节和新一局开始的功能。

### 5.2 单局战斗流程

```text
进入战斗
  -> 初始化玩家属性、武器、敌人波次
  -> 玩家移动和瞄准
  -> 武器自动攻击
  -> 敌人生成并追击玩家
  -> 击杀敌人后掉落经验/金币
  -> 玩家拾取资源并成长
  -> 时间推进后刷出更强敌人和 Boss
```

### 5.3 升级流程

```text
经验达到升级需求
  -> 暂停游戏时间
  -> 生成 3 个升级选项
  -> 玩家选择或消耗刷新次数
  -> 应用武器/被动/超进化效果
  -> 适当回复生命
  -> 恢复战斗
```

升级词条会根据当前槽位和已有技能动态生成。若主动或被动槽位已满，系统会限制新增技能，优先提供已有技能的升级，保证规则清晰。

### 5.4 结算流程

```text
玩家死亡或击败最终 Boss
  -> 停止战斗循环
  -> 播放死亡或胜利短旋律
  -> 显示结算面板
  -> 根据击杀、金币和胜利奖励结算局外金币
  -> 玩家选择重开或返回主菜单
```

## 6. 技术实现与项目亮点

### 6.1 程序化场景与验证工具

项目提供编辑器脚本用于自动生成场景、预制体和基础资源，降低手动配置遗漏的风险。核心脚本位于：

```text
Game/Assets/Editor/ZombieOverdriveSceneBuilder.cs
```

该脚本支持：

- 生成主场景。
- 生成玩家、敌人、子弹、道具、UI 等对象。
- 校验关键对象是否存在。
- 进行基础冒烟测试。

### 6.2 对象池优化

游戏中敌人、子弹、酸液弹、经验水晶、补给和黑洞等对象会频繁生成和销毁。项目使用对象池复用这些对象，减少运行时 GC 和频繁实例化带来的开销，使大量敌人同屏时仍能保持较稳定的运行。

### 6.3 闭环难度建模

本项目不是只凭感觉调整数值，而是额外编写了平衡模拟脚本。模型会考虑：

- 随机三选一升级。
- 3 主动槽和 3 被动槽限制。
- 3 次全局刷新。
- 武器 DPS、怪物血量、掉落经验之间的反馈关系。
- 玩家正常情况下获得首个超进化的时间窗口。

模型结论显示，普通玩家首个超进化大多出现在 5 分钟左右。因此游戏前 1 到 4 分钟保持较平稳，5 分钟后逐步提高敌人血量、刷怪速度和特殊敌比例，使后期不会被超进化武器完全碾压。

### 6.4 音频系统

项目没有引入外部音乐素材，而是使用代码生成简单 8bit 风格背景音乐和音效，避免版权问题。音乐包括：

- 主菜单循环音乐。
- 局内战斗循环音乐。
- Boss 阶段循环音乐。
- 死亡短旋律。
- 胜利短旋律。

主菜单和暂停菜单都提供音乐音量调节，设置会保存到本地。

### 6.5 像素风视觉表现

游戏整体采用像素风格。角色和僵尸贴图经过替换和调整，敌人根据血量有不同受伤程度表现。武器、被动、补给、金币等都拥有独立图标。特效方面尽量使用清晰的几何形状、填充色块、粒子和线条表现，符合 2D 像素游戏的简洁风格。

## 7. 游戏平衡说明

游戏平衡的目标不是让玩家始终轻松，而是让成长和压力交替出现：

- **前期**：敌人数量和血量较低，帮助玩家熟悉移动、拾取和升级。
- **中期**：玩家逐渐形成武器组合，特殊敌人和中期 Boss 开始施压。
- **超进化窗口**：玩家通过主动武器和对应被动满级获得强力武器形态。
- **后期**：敌人血量、数量、刷怪节奏和 Boss 技能增强，迫使玩家利用构筑、走位和补给资源。

局外养成会让开局更容易，但不会让整局完全失去挑战。玩家仍需要合理选择升级路线，并根据敌人类型调整移动和攻击方向。

## 8. 工程结构

项目主要目录如下：

```text
Unity大作业/
  Game/                     Unity/Tuanjie 工程根目录
    Assets/                 脚本、场景、预制体、资源
    Packages/               Unity 包配置
    ProjectSettings/        工程设置
  docs/                     文档、报告、平衡说明、演示清单
  scripts/                  构建、测试和辅助脚本
  Dist/                     最终 Windows 压缩包
```

提交工程时应保留：

```text
Game/Assets
Game/Packages
Game/ProjectSettings
```

不需要提交 Unity 自动生成的缓存目录，例如：

```text
Game/Library
Game/Temp
Game/obj
Game/Logs
Builds
```

## 10. 总结

《僵尸狂潮：超载重启》完成了一个从主菜单、局外养成、局内战斗、升级构筑、超进化、Boss 挑战到结算回流的完整单机游戏流程。项目重点不只是实现单个功能，而是将多个系统连接成可循环游玩的整体体验。

本项目的主要亮点包括：

- 完整的 2D Roguelite 生存射击核心循环。
- 6 种主动武器、12 种被动技能和 6 种超进化武器。
- 3 主动 + 3 被动的构筑限制和动态升级池。
- 局内成长与局外金币永久强化结合。
- 多类型敌人、Boss 技能、补给箱和特殊道具。
- 中文 UI、音效、8bit 背景音乐和音量调节。
- 程序化场景构建、对象池和自动验证脚本。
- 基于模拟的难度曲线调整。

整体而言，本游戏已经具备完整的玩法闭环和可交付的 Windows 运行版本，适合作为 Unity/Tuanjie 单机游戏课程大作业提交。
