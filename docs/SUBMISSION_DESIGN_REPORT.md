# 《僵尸狂潮：超载重启》设计说明

## 1. 项目基本信息

**项目名称**：僵尸狂潮：超载重启  
**开发工具**：Tuanjie / Unity  
**游戏类型**：2D 俯视角 Roguelite 生存射击  
**运行平台**：Windows 单机  
**提交内容**：Unity 游戏工程文件夹、编译后的 Windows 可执行文件、设计说明 PDF 文档

本项目是一款以“僵尸潮生存、局内成长、局外强化”为核心的单机游戏。玩家在无限循环的战场中移动、瞄准、自动攻击，击杀敌人后收集经验和金币，通过升级选择构筑武器组合，并在 Boss 压力下尽可能完成一局挑战。单局结算获得的金币会保留到局外，用于提升下一局的初始属性，形成反复挑战和逐步变强的循环。

## 2. 游戏使用说明

### 2.1 启动方式

解压提交的 Windows 游戏压缩包后，双击：

```text
ZombieOverdrive.exe
```

即可运行游戏。运行时需要保持以下文件和文件夹与 exe 位于同一目录：

```text
ZombieOverdrive_Data/
MonoBleedingEdge/
UnityPlayer.dll
UnityCrashHandler64.exe
```

### 2.2 基本操作

| 操作 | 功能 |
| --- | --- |
| `W / A / S / D` | 控制角色移动 |
| 鼠标移动 | 控制瞄准方向 |
| `ESC` | 暂停游戏，查看当前武器、被动、超进化搭配和音量设置 |
| 鼠标左键 | 点击 UI 按钮、选择升级词条 |
| 升级界面“刷新词条” | 一局内最多刷新 3 次升级选项 |

游戏采用自动攻击机制。玩家主要负责走位、拾取经验、规划升级路线和躲避敌人。

### 2.3 游戏目标

玩家需要在持续增强的僵尸潮中存活，并通过升级武器和被动技能逐步形成战斗力。游戏中会出现普通敌人、特殊敌人、中期 Boss 和最终 Boss。击败敌人可以获得经验、金币和补给资源。单局结束后，根据击杀数、局内金币和胜利状态获得局外金币。

### 2.4 菜单说明

主菜单包含“永久强化”系统。玩家可以消耗金币提升生命、移动速度、武器伤害、拾取范围、金币收益和复活能力。暂停菜单可以查看当前主动武器、被动技能、作战信息和超进化搭配，同时可以调节音乐音量、继续游戏、重开或退出到主菜单。

## 3. 游戏设计思路

### 3.1 核心体验目标

本游戏希望实现一种简单但完整的 Roguelite 生存体验：玩家初期较弱，需要通过走位和拾取经验建立第一套武器组合；中期开始出现特殊敌人和 Boss，要求玩家进行升级路线规划；后期敌人数量、血量和攻击压力提升，玩家需要依靠超进化武器、补给箱和局外养成完成挑战。

设计重点包括：

- **低操作门槛**：攻击自动进行，玩家只需移动和瞄准。
- **高成长反馈**：武器每一级升级都能带来实际效果变化。
- **组合构筑感**：主动武器与对应被动满级后可以触发超进化。
- **局内外循环**：失败也能获得金币，用于下一局永久强化。
- **清晰可展示**：所有核心系统都有 UI、特效、音效或菜单反馈。

### 3.2 核心循环

游戏的主要循环如下：

```text
进入主菜单
  -> 使用局外金币强化角色
  -> 开始战斗
  -> 移动、瞄准、自动攻击
  -> 击杀敌人并收集经验/金币
  -> 升级三选一，规划武器和被动
  -> 主动武器 + 对应被动满级后触发超进化
  -> 挑战更高强度敌人和 Boss
  -> 死亡或胜利结算
  -> 获得局外金币，回到主菜单继续强化
```

## 4. 主要系统设计

### 4.1 玩家与战斗系统

玩家角色采用 2D 俯视角移动方式，通过 `WASD` 控制移动，鼠标控制瞄准方向。游戏中会显示准心和虚线瞄准线，帮助玩家判断攻击方向。武器会自动按照冷却时间攻击，避免玩家在大量敌人围攻时需要频繁点击。

为了增强可读性，项目对不同攻击类型做了区分：

- 手枪子弹、霰弹弹丸和分裂弹使用不同形状和视觉表现。
- 电磁手套使用更粗的电弧表现链式攻击。
- 光刃拥有明显的挥剑弧线和剑阵特效。
- 黑洞使用旋转涡旋、吸积盘和吸入粒子表现范围吸附。
- 激光使用持续光束和超进化后的多线切割表现。

### 4.2 武器升级系统

游戏中共有六种主动武器。每种武器都不是单纯提升数值，而是尽量在 1 到 5 级中加入可感知的机制变化。这样玩家在升级时能明确感受到“武器形态正在成型”，而不是只看到伤害数字变大。

| 主动武器 | 基础定位 | 对应超进化被动 | 超进化形态 |
| --- | --- | --- | --- |
| 哨兵手枪 | 稳定单体输出，适合作为开局武器 | 高能推进剂 | 双持穿甲哨兵 |
| 爆裂霰弹枪 | 近距离爆发和击退，适合清理贴脸敌人 | 扩容弹药箱 | 钢雨撕裂者 |
| 电磁手套 | 链式清怪和控制，适合处理密集小怪 | 超频处理器 | 雷暴线圈 |
| 重力黑洞炮 | 范围控制和持续伤害，适合聚怪与拖延 | 重力核心 | 暗物质坍缩 |
| 光刃影切 | 近战防线和保护性输出，适合被包围时解围 | 肾上腺素 | 修罗剑阵 |
| 裂变激光枪 | 持续穿透和高线性输出，适合处理队列敌人 | 钛合金护甲 | 日冕切割束 |

#### 4.2.1 哨兵手枪

哨兵手枪是玩家开局自带武器，设计目标是“稳定、可靠、容易理解”。它前期提供基础输出，中后期通过分裂弹、双枪和穿透能力变成高频弹幕武器。

| 等级 | 具体效果 | 设计目的 |
| --- | --- | --- |
| 1 | 稳定单发瞄准射击 | 给玩家最基础的远程输出能力 |
| 2 | 射速和伤害提升 | 缓解前期击杀效率不足 |
| 3 | 命中首个目标后分裂出两发小弹 | 提供轻度群体伤害，让手枪不只打单体 |
| 4 | 变为双枪齐射 | 明显提升射击频率和视觉反馈 |
| 5 | 穿透提升，并对异常状态敌人造成双倍伤害 | 鼓励和燃烧、减速、黑洞等控制效果配合 |
| 超进化 | 多发高速穿甲弹，扇形齐射并保留分裂爆点 | 把“精准单发”强化成“高频穿透弹幕” |

#### 4.2.2 爆裂霰弹枪

爆裂霰弹枪定位为近距离爆发武器。它的强度主要来自近距离多弹丸命中，但为了避免过强，单颗弹丸伤害被控制在较低水平，依靠弹丸数量、击退和燃烧区形成特色。

| 等级 | 具体效果 | 设计目的 |
| --- | --- | --- |
| 1 | 5 发扇形弹丸，近距离击退 | 建立“贴脸爆发”定位 |
| 2 | 弹丸增至 7 发，散布更集中 | 提升有效命中率，让武器更稳定 |
| 3 | 弹丸命中后留下短暂燃烧区域 | 增加持续伤害和区域压制 |
| 4 | 装填更快，击退更强 | 增强自保能力，降低被包围风险 |
| 5 | 弹丸获得穿透，能打穿前排 | 让霰弹枪在尸潮中后期仍然有效 |
| 超进化 | 大范围爆裂弹幕，接近 360 度散射，并强化火场表现 | 从前方爆发变为全向清场工具 |

#### 4.2.3 电磁手套

电磁手套的设计目标是“链式传播”和“控制”。它前期看起来伤害不高，但等级提升后会增加链数、传导距离、眩晕、分叉和击杀爆链，适合处理密集敌群。

| 等级 | 具体效果 | 设计目的 |
| --- | --- | --- |
| 1 | 链式电弧最多命中 3 个目标 | 建立连锁攻击基础 |
| 2 | 链上限提升，首个目标承受更高伤害 | 提升主目标击杀效率 |
| 3 | 电弧有概率短暂眩晕敌人 | 增加控制反馈和生存价值 |
| 4 | 传导距离提升，并可能分叉 | 让电弧能跨越更大的敌群 |
| 5 | 击杀目标会产生电磁爆链 | 形成连锁击杀反馈 |
| 超进化 | 双主链雷暴，链数更多、分叉概率更高、减速和眩晕更明显 | 强化“雷暴扩散”的视觉和实际清怪能力 |

#### 4.2.4 重力黑洞炮

重力黑洞炮是范围控制型武器。它并不追求瞬间爆发，而是通过吸附、持续伤害和弹体吞噬改变战场结构，为其他武器创造输出环境。

| 等级 | 具体效果 | 设计目的 |
| --- | --- | --- |
| 1 | 发射牵引黑洞球 | 提供基础聚怪和持续伤害 |
| 2 | 牵引半径扩大 | 提升控制范围 |
| 3 | 黑洞附加最大生命百分比撕裂 | 让黑洞面对高血量敌人也有价值 |
| 4 | 黑洞会吞噬敌方酸弹 | 提供防御功能，克制远程敌人 |
| 5 | 同时发射双重黑洞 | 扩大控场面积 |
| 超进化 | 多重暗物质涡旋，吸附范围、持续时间和坍缩爆发增强 | 从单点控制变为大范围战场压制 |

#### 4.2.5 光刃影切

光刃影切是近战防线武器。它用于处理贴近玩家的敌人，并在高等级后提供剑气、短暂无敌、斩杀和吸血，增强玩家被包围时的反击能力。

| 等级 | 具体效果 | 设计目的 |
| --- | --- | --- |
| 1 | 前方 120 度光刃斩击 | 提供基础近战扇形攻击 |
| 2 | 斩击半径提升，并射出剑气 | 扩大安全范围，补充远端伤害 |
| 3 | 挥刀更快，挥刀瞬间有短暂无敌 | 增强操作容错和生存能力 |
| 4 | 对残血敌人造成斩杀伤害，斩 Boss 可回血 | 增加收割价值和 Boss 战续航 |
| 5 | 斩击角度扩大到大半圈 | 接近形成环绕防线 |
| 超进化 | 360 度剑阵，攻击频率更高，并拥有更明显的双层斩击特效 | 从前方防线变为全方位护身剑阵 |

#### 4.2.6 裂变激光枪

裂变激光枪是持续穿透型武器。它强调方向性输出和持续照射，适合处理排成线的敌人。升级后通过加粗、折射和热能过载提升清怪效率。

| 等级 | 具体效果 | 设计目的 |
| --- | --- | --- |
| 1 | 持续穿透激光 | 建立直线穿透输出定位 |
| 2 | 激光更粗，伤害提升 | 提升命中宽容度和基础输出 |
| 3 | 命中敌人后会折射到附近目标 | 从单线输出扩展到多目标 |
| 4 | 持续照射同一目标会热能过载 | 提供对高血量敌人的压制 |
| 5 | 折射次数提升，形成乱射光网 | 增强中后期群体清理能力 |
| 超进化 | 三线分束切割，主束更粗，折射和链爆更强 | 从单束激光进化为多线切割网络 |

### 4.3 被动技能与超进化

游戏限制为 3 个主动武器槽和 3 个被动技能槽。槽位满后，升级池会优先提供已有武器和已有被动的升级项，避免继续选择超出槽位的新技能。

游戏中共有十二种常规被动技能。被动技能既提供通用属性成长，也承担超进化配对功能。部分被动是指定武器的超进化条件，部分被动则提供经济、生存、暴击、拾取或补给方向的辅助能力。

| 被动技能 | 每级效果 | 主要定位 | 超进化搭配 |
| --- | --- | --- | --- |
| 扩容弹药箱 | 武器伤害 +7%，范围 +4% | 提升弹幕类武器的伤害和覆盖面积 | 爆裂霰弹枪 -> 钢雨撕裂者 |
| 超频处理器 | 攻击速度 +8% | 提高所有武器触发频率 | 电磁手套 -> 雷暴线圈 |
| 肾上腺素 | 移动速度 +6% | 提高走位和拉扯能力 | 光刃影切 -> 修罗剑阵 |
| 钛合金护甲 | 受到伤害 -5% | 增强生存能力 | 裂变激光枪 -> 日冕切割束 |
| 高能推进剂 | 弹体速度 +10%，等级 1/3/5 增加子弹穿透 | 强化弹道武器和穿透能力 | 哨兵手枪 -> 双持穿甲哨兵 |
| 重力核心 | 持续性效果时长 +10% | 强化黑洞、燃烧等持续区域效果 | 重力黑洞炮 -> 暗物质坍缩 |
| 超导磁铁 | 拾取范围 +0.55 | 提高经验和资源收集效率 | 通用被动 |
| 防化服 | 生命上限 +16，每秒回复 +0.28，选择时回复部分生命 | 提供血量和持续恢复 | 通用被动 |
| 贪婪芯片 | 经验获取 +6%，金币收益 +12% | 加快升级和局外收益积累 | 通用被动 |
| 战术雷达 | 暴击率 +3.5%，暴击伤害 +8% | 提供输出上限 | 通用被动 |
| 紧急除颤器 | 首次选择获得 1 次复活；之后提升升级回血 | 提高容错率 | 通用被动 |
| 战术无线电 | 补给掉落幸运 +8% | 提高补给箱和资源收益 | 通用被动 |

被动技能的设计分为三类：

- **超进化配对型**：扩容弹药箱、超频处理器、肾上腺素、钛合金护甲、高能推进剂、重力核心。它们既提供通用属性，也决定对应主动武器能否超进化。
- **生存辅助型**：防化服、紧急除颤器、钛合金护甲。它们提高生命、减伤、回血和复活能力。
- **资源成长型**：超导磁铁、贪婪芯片、战术无线电。它们强化拾取、经验、金币和补给收益。

超进化条件为：

```text
主动武器达到 5 级 + 对应被动技能达到 5 级
```

触发超进化后，武器会获得明显更强的攻击形态和特效变化。对应被动槽会被释放，但已获得的被动属性仍然保留，这样既体现了构筑奖励，也给玩家后续继续选择其他被动留下空间。

### 4.4 敌人与 Boss 系统

敌人系统包含多种类型：

| 敌人类型 | 特点 |
| --- | --- |
| 普通僵尸 | 基础近战追击敌人，用于形成主要尸潮 |
| 快速僵尸 | 移动速度更快，压迫玩家走位 |
| 喷吐僵尸 | 远程发射酸液弹，迫使玩家改变移动路线 |
| 坦克僵尸 | 血量更高，承担前排压力 |
| 中期 Boss | 具有冲锋、地裂等技能 |
| 最终 Boss | 具有血弹、酸雨、狂暴等更高压力技能 |

敌人会根据血量切换正常、受伤和濒死贴图，使玩家可以通过外观判断敌人状态。Boss 技能带有预警表现，降低“突然受伤”的挫败感。

### 4.5 经验、补给与可破坏物

敌人死亡后会掉落经验水晶和少量金币。玩家靠近经验水晶后会自动吸取，拾取范围可以通过局内被动和局外天赋提升。

地图中会生成可破坏补给箱。补给箱被武器破坏后，有概率掉落以下资源：

- **金币**：增加本局收益。
- **烤鸡**：回复生命值。
- **磁铁**：吸取全屏范围内的经验和资源。
- **炸弹**：对大范围敌人造成伤害或清场。

补给箱、炸弹和燃烧区域都使用内部填充型特效，避免只显示单一圆圈，让视觉表现更贴近像素动作游戏。

### 4.6 局外养成系统

局外养成位于主菜单“永久强化”区域。玩家可以花费金币升级六项能力：

| 永久强化 | 效果 |
| --- | --- |
| 强化体质 | 提升生命上限 |
| 机动训练 | 提升移动速度 |
| 武器校准 | 提升全武器伤害 |
| 磁场背包 | 提升初始拾取范围 |
| 收益芯片 | 提升金币收益 |
| 备用生命 | 解锁复活并提升复活后的生命 |

局外养成的设计目标是让玩家下一局开局更轻松，但不会直接破坏中后期难度。中后期通过固定时间推进的怪物血量、刷怪速度和敌人组合来维持压力。

## 5. 业务流程说明

从软件流程角度看，本游戏可以分为“主菜单流程”“单局战斗流程”“升级流程”“结算流程”四部分。

### 5.1 主菜单流程

```text
启动游戏
  -> 加载主菜单
  -> 读取局外金币和天赋等级
  -> 玩家选择永久强化或直接开始
  -> 点击开始战斗
  -> 进入局内状态
```

主菜单不仅是入口页面，也承担局外养成、音量调节和新一局开始的功能。

### 5.2 单局战斗流程

```text
进入战斗
  -> 初始化玩家属性、武器、敌人波次
  -> 玩家移动和瞄准
  -> 武器自动攻击
  -> 敌人生成并追击玩家
  -> 击杀敌人后掉落经验/金币
  -> 玩家拾取资源并成长
  -> 时间推进后刷出更强敌人和 Boss
```

### 5.3 升级流程

```text
经验达到升级需求
  -> 暂停游戏时间
  -> 生成 3 个升级选项
  -> 玩家选择或消耗刷新次数
  -> 应用武器/被动/超进化效果
  -> 适当回复生命
  -> 恢复战斗
```

升级词条会根据当前槽位和已有技能动态生成。若主动或被动槽位已满，系统会限制新增技能，优先提供已有技能的升级，保证规则清晰。

### 5.4 结算流程

```text
玩家死亡或击败最终 Boss
  -> 停止战斗循环
  -> 播放死亡或胜利短旋律
  -> 显示结算面板
  -> 根据击杀、金币和胜利奖励结算局外金币
  -> 玩家选择重开或返回主菜单
```

## 6. 技术实现与项目亮点

### 6.1 程序化场景与验证工具

项目提供编辑器脚本用于自动生成场景、预制体和基础资源，降低手动配置遗漏的风险。核心脚本位于：

```text
Game/Assets/Editor/ZombieOverdriveSceneBuilder.cs
```

该脚本支持：

- 生成主场景。
- 生成玩家、敌人、子弹、道具、UI 等对象。
- 校验关键对象是否存在。
- 进行基础冒烟测试。

### 6.2 对象池优化

游戏中敌人、子弹、酸液弹、经验水晶、补给和黑洞等对象会频繁生成和销毁。项目使用对象池复用这些对象，减少运行时 GC 和频繁实例化带来的开销，使大量敌人同屏时仍能保持较稳定的运行。

### 6.3 闭环难度建模

本项目不是只凭感觉调整数值，而是额外编写了闭环 Monte Carlo 平衡模拟脚本：

```text
scripts/balance-model.ps1
```

这个脚本的目的不是追求完全精确的物理级模拟，而是用一个近似模型回答几个关键设计问题：

- 普通玩家大约什么时候能获得首个超进化？
- 武器 DPS 增长是否会让后期敌人完全失去压力？
- 敌人血量、刷怪速度和经验掉落之间是否形成了合理反馈？
- 局外养成让开局变容易后，中后期是否仍有挑战？

需要说明的是，当前这组模拟结果采用的是 **0 局外养成的新号基准**，即主菜单中的生命、移速、伤害、拾取范围、金币收益和复活天赋都按 0 级处理。这样做的原因是：如果 0 养成状态下曲线已经完全失去压力，那么后续局外成长会让游戏过早变成无挑战状态。因此模型首先保证“新号也能进入中期并看到构筑成型，但不保证第一局稳定通关”。

局外养成在设计上承担的是“降低开局压力、提高通关稳定性”的作用，而不是让玩家第一局就必定通关。随着玩家投入金币强化生命、伤害、移速和复活能力，前期容错率和清怪效率会明显提高，通关率也会随之上升。

模型会考虑：

- 随机三选一升级。
- 3 主动槽和 3 被动槽限制。
- 3 次全局刷新。
- 武器 DPS、怪物血量、掉落经验之间的反馈关系。
- 玩家正常情况下获得首个超进化的时间窗口。

#### 6.3.1 闭环反馈关系

Roguelite 生存游戏的难度不是单向变化的。武器变强会提高击杀速度，击杀速度又会提高经验获取速度，经验获取速度会带来更多升级，最终继续提高武器 DPS。因此本项目把难度看成一个反馈系统：

```text
当前武器和被动等级
  -> 估算当前构筑 DPS
  -> 与当前时间段敌人平均血量比较
  -> 得到单位时间击杀率
  -> 根据击杀率和经验掉落计算经验增长
  -> 经验达到阈值后触发升级选择
  -> 更新武器/被动/超进化状态
  -> 下一秒重新计算 DPS 和压力
```

模型每一秒都会重新计算一次当前构筑的输出能力和敌人压力，避免把玩家成长看成固定曲线。

#### 6.3.2 主要变量

模型中使用的核心变量如下：

| 变量 | 含义 |
| --- | --- |
| `DPS(t)` | 当前时间玩家构筑的估算输出 |
| `HP_enemy(t)` | 当前波次敌人的平均血量 |
| `SpawnRate(t)` | 当前每秒刷怪数量 |
| `XP_enemy(t)` | 当前敌人的平均经验价值 |
| `KillRate(t)` | 每秒击杀数量，受 DPS 和刷怪速度共同限制 |
| `XP(t)` | 当前累计经验 |
| `Level(t)` | 当前等级 |
| `CombatPressure(t)` | 敌人血量吞吐量与玩家 DPS 的比值 |

其中比较重要的是 `CombatPressure`：

```text
CombatPressure(t) = SpawnRate(t) * HP_enemy(t) / DPS(t)
```

这个值可以理解为“当前刷出来的敌人血量压力是否超过玩家输出能力”。如果数值接近 1，说明刷怪压力和玩家输出大致匹配；如果明显小于 1，玩家可以较轻松清场；如果大于 1，敌人会逐渐积压，玩家需要走位、补给或更强构筑来缓解压力。

#### 6.3.3 升级选择模拟

玩家升级选择并不是按最优解固定选择，而是模拟“普通玩家”的选择行为：

- 每次升级生成 3 个候选项。
- 玩家倾向于选择已有武器、已有被动和能形成超进化搭配的选项。
- 仍然保留一定随机性，避免假设玩家每次都完美选择。
- 一局内最多允许 3 次刷新词条。
- 超进化必须满足“主动武器 5 级 + 对应被动 5 级”。

这种处理比“假设玩家 2 到 3 分钟必定做出超武”更合理。因为真实情况下，玩家需要至少 10 次相关升级，还必须在随机词条中同时抽到目标主动和对应被动。模拟结果显示，大部分普通局的首个超进化不会过早出现。

#### 6.3.4 模拟结论

模型使用 1200 局 Monte Carlo 采样，对比了旧曲线和当前曲线：

| 曲线 | 最终等级 p25/中位数/p75 | 击杀中位数 | 首个超进化 p25/中位数/p75 | 无超进化比例 |
| --- | --- | --- | --- | --- |
| 旧曲线 | 34 / 38 / 39 | 2805 | 04:48 / 06:02 / 06:03 | 1.5% |
| 当前曲线 | 34 / 40 / 42 | 2758 | 04:47 / 05:39 / 06:02 | 1.6% |

结论是：普通玩家首个超进化大多出现在 5 分钟左右，而不是 2 到 3 分钟。因此难度曲线没有在开局过早加压，而是在 5 分钟之后逐步提高中后期压力。

由于这组结果基于 **0 局外养成**，它更适合作为“最低成长状态”的难度基准。实际游玩中，玩家通过几局失败或半成功结算获得金币后，会逐渐提高永久强化等级，后续局的前期体验会比模型基准更轻松。这也符合本项目“失败积累资源 -> 局外强化 -> 更接近通关”的 Roguelite 设计目标。

#### 6.3.5 压力曲线调整

模型还记录了每分钟的 `CombatPressure` 中位数：

| 分钟 | 旧曲线压力中位数 | 当前曲线压力中位数 | 调整目的 |
| --- | --- | --- | --- |
| 1 | 0.43 | 0.44 | 基本不变，让玩家熟悉移动和拾取 |
| 2 | 0.35 | 0.36 | 基本不变，保证早期可成型 |
| 3 | 0.40 | 0.43 | 轻微增加，开始引入压力 |
| 4 | 0.38 | 0.42 | 轻微增加，为中期过渡做准备 |
| 5 | 0.66 | 0.75 | 开始进入超进化窗口，提高敌人压力 |
| 6 | 0.68 | 0.64 | 保留获得超进化后的短暂强势期 |
| 7 | 0.71 | 0.78 | 重新抬升压力 |
| 8 | 0.64 | 0.93 | 明显增强后期刷怪压力 |
| 9 | 0.98 | 1.31 | 让后期敌人开始积压 |
| 10 | 0.98 | 1.40 | 保证最终阶段有挑战 |

最终调整策略是：

- 前 1 到 4 分钟尽量保持稳定，不强行压制玩家。
- 5 分钟后增加敌人血量成长和特殊敌比例。
- 8 到 10 分钟显著提高刷怪压力，避免超进化武器完全碾压。
- 不根据“玩家是否拿到超武”动态增强怪物，避免让玩家感觉被系统惩罚。
- 使用固定时间推进的难度曲线，让玩家能感受到明确的阶段变化。

这套建模的亮点在于，它把“武器 DPS -> 击杀率 -> 经验获取 -> 等级提升 -> DPS 再增长”的循环纳入考虑，使数值调整更符合实际游玩过程。

游戏整体采用像素风格。角色和僵尸贴图经过替换和调整，敌人根据血量有不同受伤程度表现。武器、被动、补给、金币等都拥有独立图标。特效方面尽量使用清晰的几何形状、填充色块、粒子和线条表现，符合 2D 像素游戏的简洁风格。

## 7. 游戏平衡说明

游戏平衡的目标不是让玩家始终轻松，而是让成长和压力交替出现：

- **前期**：敌人数量和血量较低，帮助玩家熟悉移动、拾取和升级。
- **中期**：玩家逐渐形成武器组合，特殊敌人和中期 Boss 开始施压。
- **超进化窗口**：玩家通过主动武器和对应被动满级获得强力武器形态。
- **后期**：敌人血量、数量、刷怪节奏和 Boss 技能增强，迫使玩家利用构筑、走位和补给资源。

局外养成会让开局更容易，但不会让整局完全失去挑战。玩家仍需要合理选择升级路线，并根据敌人类型调整移动和攻击方向。

本项目最终采用的平衡思路可以概括为：

```text
开局宽松 -> 中期成型 -> 超进化爆发 -> 后期重新施压
```

这样的曲线能保证玩家获得超进化时有明显爽感，但不会让后半局完全没有威胁。由于模型显示首个超进化的中位时间约在 5 分 39 秒，因此游戏把更明显的血量成长、刷怪加速和坦克/远程敌比例提升放在 5 分钟之后。这样既符合玩家自然成长节奏，也能让 Boss 和后期尸潮继续承担挑战作用。

从通关预期上看，本项目并不要求 0 局外养成的第一局就能稳定通关。0 养成状态更像是新玩家基准体验：玩家能够熟悉移动、升级、超进化和 Boss 压力，但通关难度较高。随着局外金币投入到永久强化中，开局血量、伤害、移速、拾取范围和复活能力逐渐提高，玩家会更稳定地进入后期并挑战最终 Boss。这种设计让“失败”也能转化为下一局的成长资源，强化 Roguelite 的反复挑战体验。

## 10. 总结

《僵尸狂潮：超载重启》完成了一个从主菜单、局外养成、局内战斗、升级构筑、超进化、Boss 挑战到结算回流的完整单机游戏流程。项目重点不只是实现单个功能，而是将多个系统连接成可循环游玩的整体体验。

本项目的主要亮点包括：

- 完整的 2D Roguelite 生存射击核心循环。

整体而言，本游戏已经具备完整的玩法闭环和可交付的 Windows 运行版本，适合作为 Unity/Tuanjie 单机游戏课程大作业提交。
